## 2. Create `src/dom_xss_auditor.py` (The Safe DOM XSS Tester)

This module tests for DOM-based XSS using ONLY non-malicious, safe payloads.

### Class: `DOMXSSAuditor`

**Safe Payloads (NON-MALICIOUS - these do NOT execute harmful code):**
```python
SAFE_DOM_PROBES = [
    # Math-based probes (harmless, detectable)
    ("{{7*7}}", "49", "Template injection probe"),
    ("${7*7}", "49", "EL injection probe"),
    
    # Visual-only probes (render but don't execute)
    ("<marquee>DOMTEST</marquee>", "DOMTEST", "Marquee visual probe"),
    ("<b>DOMBOLD</b>", "DOMBOLD", "Bold tag reflection probe"),
    ("<h1>DOMH1</h1>", "DOMH1", "H1 tag reflection probe"),
    
    # Attribute probes (detect unsafe attribute injection)
    ('" data-testid="domprobe', "domprobe", "Attribute breakout probe"),
    ("' data-domtest='1", "domtest", "Single-quote attribute probe"),
    
    # Event handler probes (detect if event handlers are attached)
    # NOTE: These use console.log, NOT alert(), to avoid disrupting the UI
    ('" onfocus="console.log(42)" autofocus="', "console.log", "Event handler probe"),
    
    # URL-based probes (for href/src attributes)
    ("javascript:void(0)", "javascript:", "JS protocol probe"),
]
```

**Method: `audit_page(page, input_fields: list[dict]) -> list[DOMXSSResult]`**
1. For each input field discovered by the AI Explorer:
   a. For each safe probe in `SAFE_DOM_PROBES`:
      - Clear the input field
      - Type the probe payload using `page.fill()` or `page.type()`
      - Submit the form or trigger the relevant event (blur, change, keyup)
      - Wait 2 seconds for DOM updates
      - **DOM Check (Local - 0 tokens):** Use `page.evaluate()` to check:
        ```javascript
        // Check if payload was rendered unsanitized in the DOM
        document.body.innerHTML.includes('PAYLOAD_HERE')
        ```
      - **Execution Check (Local - 0 tokens):** Use `page.evaluate()` to check:
        ```javascript
        // Check if a <marquee>, <b>, or <h1> tag was actually created
        document.querySelectorAll('marquee, b, h1').length > 0
        ```
      - If the DOM check OR execution check returns true:
        - Take a screenshot
        - Mark as `POTENTIAL_DOM_XSS`
        - Send screenshot + DOM snippet to Vision LLM for verification:
          > "Analyze this screenshot. The scanner injected this safe probe: [PAYLOAD]. 
          > Can you visually confirm the probe was rendered unsanitized in the DOM?
          > Look for: rendered HTML tags, visible probe text, broken UI elements.
          > Return JSON: {dom_xss_confirmed: bool, visual_evidence: '...', confidence: 0.0-1.0}"
2. Return all `DOMXSSResult` objects

**Method: `audit_url_params(page, current_url)`**
1. Extract all URL query parameters
2. For each parameter, append a safe probe to the URL
3. Navigate to the modified URL
4. Check `document.body.innerHTML` for reflection
5. Same LLM verification flow as above

### 💡 Why This Architecture is Revolutionary:

#### 1. **It Discovers What HAR Files Miss**
HAR files only capture what the human tester clicked during recording. If they forgot to click the "Admin Settings" tab, the HAR file has no record of it. The AI Explorer **autonomously finds every tab, link, and hidden page** by visually analyzing the navigation.

#### 2. **It Understands Context, Not Just Endpoints**
Traditional scanners see `/services/data/v58.0/sobjects/Account/001XX000003DGbY`. The AI Explorer sees *"This is the Account Details page for Acme Corp, showing the billing address, contact list, and a 'Delete Account' button that shouldn't be visible to standard users."*

#### 3. **Role Diffing Finds Privilege Escalation**
By running the explorer as both Admin and Standard User, the LLM can say: *"The Standard User can see the 'Export All Data' button on the Reports page. This should be Admin-only. This is a Broken Function Level Authorization (API5) vulnerability."*

#### 4. **DOM XSS Detection is Actually Reliable**
Instead of guessing if a payload executed by looking at HTTP responses, the tool:
- Injects `<marquee>DOMTEST</marquee>` into a search field
- Uses `page.evaluate()` to check if a `<marquee>` tag actually exists in the live DOM
- Takes a screenshot and asks the Vision LLM: *"Do you see scrolling text that says 'DOMTEST'?"*
- This is **triple-verified**: DOM check + Visual check + LLM reasoning

#### 5. **Safe Probes = Zero Risk**
By using `<marquee>`, `<b>`, and `{{7*7}}` instead of `<script>alert()</script>`, you can run this against **production environments** (with permission) without any risk of disrupting users or triggering WAF blocks.
